# Glossary

## Columns

- **orchestration_model**: how the harness sequences agent work.
  - `state-machine` ‚Äî explicit states and transitions
  - `dag` ‚Äî directed acyclic graph of steps
  - `event-driven` ‚Äî agents react to events/messages
  - `turn-based` ‚Äî linear request/response turns
  - `actor-based` ‚Äî agents as independent actors
  - `conversation-graph` ‚Äî graph of conversation nodes
  - `other`
- **tool_integration**: how tools are exposed to agents.
  - `native-tool-calling` ‚Äî LLM-native function calling
  - `mcp` ‚Äî Model Context Protocol
  - `a2a` ‚Äî Agent-to-Agent protocol
  - `custom-sdk` ‚Äî framework-specific tool API
  - `plugin-api` ‚Äî generic plugin mechanism
  - `none`
- **memory_model**: how context/state persists.
  - `conversation-only` ‚Äî only in-context messages
  - `persistent-state` ‚Äî durable state across sessions
  - `shared-memory` ‚Äî memory shared across agents
  - `vector-db` ‚Äî retrieval-augmented memory
  - `none`
- **multi_agent_primitives**: built-in ways for multiple agents to coordinate.
  - `messaging` ‚Äî direct agent messages
  - `hierarchy` ‚Äî supervisor/worker relationships
  - `roles` ‚Äî role-based delegation
  - `swarm` ‚Äî many simple agents
  - `handoffs` ‚Äî explicit task transfer
  - `none`
- **human_in_the_loop**: built-in mechanisms for human intervention.
  - `breakpoints` ‚Äî pause before key steps
  - `approvals` ‚Äî require human approval for actions
  - `feedback` ‚Äî capture human corrections
  - `none`
- **observability**: built-in monitoring/telemetry.
  - `tracing` ‚Äî distributed/execution tracing
  - `cost-tracking` ‚Äî token/cost tracking
  - `token-usage` ‚Äî per-call token counts
  - `logging` ‚Äî structured logging
  - `none`
- **guardrails**: safety/limits.
  - `sandboxing` ‚Äî execution isolation
  - `output-moderation` ‚Äî content filtering
  - `rate-limits` ‚Äî request/throttling limits
  - `turn-caps` ‚Äî max execution turns
  - `none`
- **deployment_mode**: typical deployment shape.
  - `library` ‚Äî importable package
  - `api-server` ‚Äî self-hosted HTTP service
  - `container` ‚Äî packaged container image
  - `cloud-hosted` ‚Äî vendor-managed service
  - `embedded` ‚Äî IDE/editor plugin
  - `other`
